外设与通信协议

掌握 GPIO、I2C、SPI、I2S、SAI 等外设操作。理解时钟树配置。重点攻克音频接口：I2S/PDM/TDM。

2.1 GPIO — 万物之始

GPIO（通用输入输出）是控制外部硬件的基础。音频设备中 GPIO 用于：Codec 芯片复位、LED 指示、按键输入、Mute 控制等。

gpio.c — 寄存器级 GPIO 操作 
// STM32F4 GPIO 寄存器结构 (AHB1 总线基地址 0x4002_0000)
typedef struct {
    volatile uint32_t MODER;    // 模式: 00=输入, 01=输出, 10=复用, 11=模拟
    volatile uint32_t OTYPER;   // 输出类型: 0=推挽, 1=开漏
    volatile uint32_t OSPEEDR;  // 输出速度: 00=低, 01=中, 10=高, 11=极高
    volatile uint32_t PUPDR;    // 上下拉: 00=无, 01=上拉, 10=下拉
    volatile uint32_t IDR;      // 输入数据 (只读)
    volatile uint32_t ODR;      // 输出数据 (读写)
    volatile uint32_t BSRR;     // 置位/复位 (写 1 有效)
    volatile uint32_t LCKR;     // 配置锁定
    volatile uint32_t AFR[2];   // 复用功能选择 (AF0-AF15)
} GPIO_TypeDef;

#define GPIOA ((GPIO_TypeDef *)0x40020000)

// 配置 PA4 为 Codec 复位引脚（推挽输出）
void codec_reset_pin_init(void) {
    // ① 使能 GPIOA 时钟 — 不开时钟寄存器不工作！
    RCC->AHB1ENR |= RCC_AHB1ENR_GPIOAEN;

    // ② 配置 PA4 为输出模式
    GPIOA->MODER &= ~(3U << (4 * 2));   // 清除位 [9:8]
    GPIOA->MODER |=  (1U << (4 * 2));   // 设为 01 = 通用输出

    // ③ 推挽输出，高速
    GPIOA->OTYPER &= ~(1U << 4);        // 0 = 推挽
    GPIOA->OSPEEDR |= (3U << (4 * 2));  // 11 = 极高速 (100MHz)

    // ④ 无上下拉
    GPIOA->PUPDR &= ~(3U << (4 * 2));
}

// 复位 Codec 芯片
void codec_reset(void) {
    GPIOA->BSRR = (1U << (4 + 16));  // PA4 = LOW (复位有效)
    delay_ms(5);                      // 保持 5ms
    GPIOA->BSRR = (1U << 4);         // PA4 = HIGH (释放复位)
    delay_ms(10);                     // 等待 Codec 稳定
}
① 时钟使能是第一件事。STM32 的每个外设都有独立的时钟门控位。忘记开时钟是最常见的错误 — 寄存器写了但没效果，读回来全是 0。
② BSRR 寄存器是原子操作：低 16 位置位，高 16 位复位。不能用 ODR 做 |= 操作（非原子，会被中断打断）。
可淡化

位操作样板 AI 写得又快又对

GPIO 翻转、Mute 控制这些 MODER/BSRR 位操作是标准样板。描述清楚"哪个 pin 做什么"，AI 生成代码又快又对。你的精力放在理解 pin mux 的全局规划上。

2.2 I2C — Codec 配置总线

I2C 用于配置音频 Codec 芯片的参数（采样率、增益、滤波器等）。不是传输音频数据 — 那是 I2S/SAI 的工作。

sequenceDiagram
    participant MCU as MCU (I2C Master)
    participant Codec as Codec 芯片 (I2C Slave)
    Note over MCU,Codec: 配置 WM8994 Codec 采样率
    MCU->>Codec: START + 0x34 (Codec I2C 地址 + W)
    Codec-->>MCU: ACK
    MCU->>Codec: 0x01 (寄存器地址: PLL_CONFIG)
    Codec-->>MCU: ACK
    MCU->>Codec: 0x08 (寄存器值: PLL 使能 + 分频)
    Codec-->>MCU: ACK
    MCU->>Codec: STOP

I2C 特点

  • • 两线：SDA（数据）+ SCL（时钟）
  • • 速度：标准 100kHz / 快速 400kHz
  • • 只传控制命令，不传音频
  • • 典型用途：Codec 初始化、音量调节

音频 Codec 的 I2C 用法

  • • 上电后通过 I2C 配置 PLL/分频器
  • • 设置采样率（48kHz / 16kHz / 8kHz）
  • • 配置输入/输出增益
  • • 使能/禁用 DSP 模块

2.3 SPI — 高速数据传输

SPI 用于驱动显示屏（LVGL UI）、Flash 存储、传感器等。全双工，速度可达数十 MHz。在音频智能眼镜中，SPI 可能连接显示驱动和 IMU 传感器。

spi_display.c — SPI 驱动 LCD 显示 
// SPI 配置 — 驱动 ST7789 LCD (InfiniTime 项目实际使用)
void spi_lcd_init(void) {
    SPI_HandleTypeDef hspi;
    hspi.Instance = SPI1;
    hspi.Init.Mode = SPI_MODE_MASTER;         // MCU 是主设备
    hspi.Init.Direction = SPI_DIRECTION_2LINES; // 全双工
    hspi.Init.DataSize = SPI_DATASIZE_8BIT;   // 8 位数据
    hspi.Init.CLKPolarity = SPI_POLARITY_LOW; // CPOL=0
    hspi.Init.CLKPhase = SPI_PHASE_1EDGE;     // CPHA=0 (Mode 0)
    hspi.Init.NSS = SPI_NSS_SOFT;             // 软件片选
    hspi.Init.BaudRatePrescaler = SPI_BAUDRATEPRESCALER_2; // 32MHz
    hspi.Init.FirstBit = SPI_FIRSTBIT_MSB;
    HAL_SPI_Init(&hspi);
}

// 通过 SPI 发送显示数据（DMA 模式，不阻塞 CPU）
void spi_lcd_send_dma(const uint8_t *data, uint16_t len) {
    // 拉低 DC 引脚 = 数据模式
    HAL_GPIO_WritePin(LCD_DC_GPIO_Port, LCD_DC_Pin, GPIO_PIN_SET);
    // 拉低 CS 片选
    HAL_GPIO_WritePin(LCD_CS_GPIO_Port, LCD_CS_Pin, GPIO_PIN_RESET);

    // DMA 发送 — CPU 可以同时做其他事
    HAL_SPI_Transmit_DMA(&hspi1, (uint8_t*)data, len);
}

实际项目：InfiniTime 项目用 SPI 驱动 ST7789 显示屏（240×240）。结合 DMA 实现无阻塞刷新。阅读它的 St7789.cpp 可以看到完整的 SPI LCD 驱动。

可淡化

显示驱动初始化是标准样板

ST7789 等 LCD 驱动的 SPI 初始化序列高度标准化。CubeMX 或 AI 都能生成。值得理解的是 DMA 传输模式对帧率的影响。

2.4 I2S / SAI — 音频数据核心接口

sai_audio.c — SAI 配置为 I2S 主模式 
// SAI 配置 — I2S 主模式发送 (播放音频到 Codec)
void sai_i2s_master_tx_init(void) {
    SAI_HandleTypeDef hsai;
    hsai.Instance = SAI1_Block_A;

    // ① 音频模式：主发送 (MCU 提供时钟)
    hsai.Init.AudioMode = SAI_MODEMASTER_TX;

    // ② 同步模式：异步 (独立时钟)
    hsai.Init.Synchro = SAI_ASYNCHRONOUS;

    // ③ 时钟配置
    hsai.Init.OutputDrive = SAI_OUTPUTDRIVE_ENABLE;
    hsai.Init.NoDivider = SAI_MASTERDIVIDER_ENABLE;

    // ④ I2S 协议参数
    SAI_InitTypeDef proto = {0};
    proto.Protocol = SAI_I2S_STANDARD;       // I2S 飞利浦标准
    proto.DataSize = SAI_DATASIZE_24BIT;     // 24-bit 音频数据
    proto.FirstBit = SAI_FIRSTBIT_MSB;       // MSB 优先
    proto.FrameSize = SAI_FRAMESIZE_32;      // 32-bit 帧 (24bit 数据 + 8bit 填充)
    proto.SlotActive = SAI_SLOTACTIVE_0 | SAI_SLOTACTIVE_1; // 左右声道

    // ⑤ 位时钟 = 64 × FS
    // 对于 48kHz: BCLK = 64 × 48000 = 3.072 MHz
    // MCLK = 256 × 48000 = 12.288 MHz (Codec 通常需要这个)

    HAL_SAI_Init(&hsai);
}

// 启动 DMA 音频播放
HAL_StatusTypeDef audio_play_start(int16_t *buffer, uint32_t size) {
    // buffer 是 DMA 双缓冲区，size 是总大小
    // DMA 半传输中断会在 buffer[size/2] 传输完成时触发
    return HAL_SAI_Transmit_DMA(&hsai, (uint8_t*)buffer, size);
}
① Master TX = MCU 提供时钟并发送数据。Slave RX = Codec 接收。这意味着 MCU 控制 BCLK 和 WS 的频率。
② 帧大小 32-bit 是标准配置：I2S 传输 24-bit 数据在一个 32-bit 帧中，剩余 8 位是填充。这样 BCLK = 64 × FS（每声道 32 个 BCLK 周期）。
③ 时钟链：MCLK → Codec 内部 PLL → BCLK/WS。STM32 的 SAI 时钟来自 PLLI2S（专门的音频 PLL），可以精确产生 12.288MHz 等音频常用频率。
不可替代

时钟数学必须内化

256×FS = 12.288MHz——为什么标准 PLL 无法精确产生这个频率？时钟关系算错 = Codec 静默，不报任何异常。

2.5 PDM — 数字麦克风接口

MEMS 数字麦克风输出 PDM（脉冲密度调制）信号。需要硬件滤波器或 CMSIS-DSP 的 PDM 转 PCM 库将其转换为标准 PCM 音频数据。智能眼镜的麦克风通常用 PDM。

pdm_mic.c — PDM 麦克风配置 
// PDM 麦克风通过 SPI/I2S 外设接收数据
// 典型配置：
//   - PDM 时钟: 1.024 - 3.072 MHz (由 MCU 提供)
//   - 输出采样率: 16kHz (常用语音频率)
//   - 抽取比: 64 (PDM_CLK / FS)

void pdm_mic_init(void) {
    // 配置 SPI/I2S 为 PDM 接收模式
    I2S_HandleTypeDef hi2s;
    hi2s.Instance = SPI2;
    hi2s.Init.Mode = I2S_MODE_MASTER_RX;    // 主接收
    hi2s.Init.Standard = I2S_STANDARD_PDM;   // PDM 模式
    hi2s.Init.DataFormat = I2S_DATAFORMAT_16B;
    hi2s.Init.MCLKOutput = I2S_MCLKOUTPUT_DISABLE;
    hi2s.Init.AudioFreq = I2S_AUDIOFREQ_1024K; // PDM 时钟 ~1MHz
    HAL_I2S_Init(&hi2s);
}

// PDM → PCM 转换 (使用 CMSIS-DSP PDM 库)
#include "arm_pdm_fir_init.h"

void pdm_to_pcm(uint8_t *pdm_buf, int16_t *pcm_out) {
    // CMSIS-DSP 提供 PDM 滤波器
    // 将高频 PDM 位流转换为低频 PCM 样本
    // 抽取比 = 64 意味着每 64 个 PDM 位 → 1 个 PCM 样本
    arm_pdm_fir_to_pcm(&pdm_filter, pdm_buf, pcm_out, pcm_buf_size);
}

智能眼镜中的麦克风：典型的助听器/智能眼镜用 2-4 个 MEMS PDM 麦克风做波束成形和降噪。nRF5340 有硬件 PDM 外设，可以直接接收 PDM 数据。

Decimation 比率影响音质

PDM 时钟 1MHz decimation 64 倍 = 15.6kHz 有效带宽。这个比率决定了你的麦克风能录到什么。这是信号处理的物理约束。

2.6 时钟树 — 音频的心跳

音频系统对时钟精度极其敏感。48kHz 采样率意味着每 20.83μs 必须精确地处理一个采样。时钟树的配置直接决定音频质量。

为什么需要专门的音频 PLL？标准 PLL 产生 168MHz 系统时钟，但 48kHz × 256 = 12.288MHz 无法从 168MHz 整数分频得到。STM32 提供独立的 PLLI2S，可以灵活配置乘法因子 N 和分频因子 R 来精确产生音频主时钟。

不可替代

PLLI2S 分频比必须一次算对

N/M/R 三个参数需要整数运算精确匹配目标频率。算错 = Codec 静默，不报错不 HardFault，就是没声音。这种 bug AI 帮不了你调试。
