术语速查

术语速查

音频嵌入式开发中常见的术语，一句话解释 + 在哪个阶段学到。

ADC / DAC

模数转换器 / 数模转换器。ADC 把模拟音频（麦克风信号）转为数字，DAC 把数字音频转为模拟（驱动扬声器）。通常集成在 Codec 芯片中。

阶段二：外设与通信协议

ANC (Active Noise Cancellation)

主动降噪。用参考麦克风采集环境噪声，生成反相信号叠加到输出，实现降噪。Feedforward ANC 最简单，Hybrid ANC 效果最好。

阶段五：音频 DSP

Biquad

双二阶（Bi-quadratic）IIR 滤波器。音频 EQ 的基本单元。每个 EQ 频段 = 一个 Biquad。CMSIS-DSP 提供 arm_biquad_cascade_df1_f32 优化实现。

阶段五：音频 DSP

BLE LE Audio / LC3

蓝牙低功耗音频新标准。LC3 是其专用编解码器，同码率质量比 SBC 好 50%。ISO 通道保证时序同步。帧长 7.5-20ms。

nRF5340 Audio 源码走读

ESP32-S3

Espressif 的 Xtensa LX7 双核 MCU，240MHz + 8MB PSRAM + WiFi/BLE。虽非 Cortex-M，但音频管线（I2S PDM、Opus 编码、BLE 传输）与 Cortex-M 项目高度重叠，是优秀的参考对比平台。

omiGlass 专题走读

Opus

开源音频编解码器，6-510kbps 可调。支持语音（VOIP 模式）和音乐。16kHz/32kbps/20ms 配置下语音质量接近原始 PCM。比 LC3 更通用，但不如 LC3 适合 BLE 专用场景。

omiGlass 专题走读

μ-law (mu-law)

16-bit PCM → 8-bit 的对数压缩编码（2:1 压缩）。64kbps @ 8kHz。北美电话标准。已被 Opus/LC3 等现代编解码器取代，但实现简单（查表法），适合理解音频编码基础。

omiGlass 专题走读

.bss / .data 段

.bss = 未初始化的全局变量（启动时清零），.data = 已初始化的全局变量（从 Flash 复制到 RAM）。startup 代码负责处理这两个段。

阶段一：Cortex-M 架构基础

CMSIS

Cortex Microcontroller Software Interface Standard。ARM 定义的软件接口标准。CMSIS-DSP 是其 DSP 库，CMSIS-RTOS2 是 RTOS 抽象层。

阶段五：音频 DSP

Codec (音频芯片)

Coder-Decoder 芯片。集成 ADC + DAC + 放大器 + DSP。MCU 通过 I2C 配置它（采样率、增益），通过 I2S 传输音频数据。常见：WM8731, AK4556, CS43L22。

阶段二：外设与通信协议

DMA (Direct Memory Access)

直接内存访问。硬件模块，无需 CPU 干预即可在内存和外设之间搬运数据。音频系统的核心 — DMA 自动搬运音频数据，CPU 专注 DSP。

阶段三：DMA 数据引擎

DRC (Dynamic Range Compression)

动态范围压缩。自动控制音量 — 声音太大时压低，太小时提升。参数：阈值、压缩比、启动时间、释放时间。

阶段五：音频 DSP

DWT (Data Watchpoint and Trace)

Cortex-M 内置调试外设。周期计数器（CYCCNT）可以精确测量代码执行时间（单周期精度）。还有数据观察点和 ITM 跟踪功能。

阶段六：系统级工程

FPU (Floating Point Unit)

浮点运算单元。M4F 及以上型号的硬件浮点加速器。单精度浮点乘法 1 周期（~6ns @ 168MHz），比软件模拟快 10 倍。音频 DSP 必需。

阶段一：Cortex-M 架构基础

HardFault

硬故障。Cortex-M 最常见的崩溃类型。原因：空指针、栈溢出、访问未映射内存。排查：读 SCB->HFSR/CFSR 寄存器和栈帧中的 PC 值。

阶段一：Cortex-M 架构基础

I2C

两线串行总线（SDA + SCL）。用于配置外设芯片（如 Codec 的采样率、增益）。速度：标准 100kHz / 快速 400kHz。不传音频数据。

阶段二：外设与通信协议

I2S / SAI

Inter-IC Sound / Serial Audio Interface。数字音频传输标准。SAI 是 STM32 的 I2S 增强版。信号线：SCK（位时钟）、WS（左右声道）、SD（数据）、MCLK（主时钟）。

阶段二：外设与通信协议

NVIC (Nested Vectored Interrupt Controller)

嵌套向量中断控制器。Cortex-M 内置硬件。支持中断嵌套、尾链（零延迟切换）、优先级分组。音频中断（DMA/SAI）优先级必须设高。

阶段一：Cortex-M 架构基础

OTA / DFU / Bootloader

OTA = 空中固件升级，DFU = Device Firmware Update，Bootloader = 引导加载程序。通常用双 Bank Flash 方案：新固件写入备用 Bank，验证后交换。

阶段六：系统级工程

PDM (Pulse Density Modulation)

脉冲密度调制。MEMS 数字麦克风的输出格式。需要低通滤波抽取转换为 PCM。nRF5340 有硬件 PDM 外设。

阶段二：外设与通信协议

Ping-Pong 双缓冲

音频 DMA 的核心模式。DMA 和 CPU 交替使用两块缓冲区（Ping 和 Pong），通过半传输/全传输中断切换。实现无间断音频流。

阶段三：DMA 数据引擎

SVC / PendSV / SysTick

FreeRTOS 的三个关键中断：SVC 用于启动第一个任务，PendSV 用于任务间上下文切换（最低优先级保证安全），SysTick 提供 1ms 时基。

阶段四：RTOS 实时操作系统

TDM (Time Division Multiplexing)

时分复用。I2S 的多通道扩展。在一个 I2S 数据线上通过时分传输 4-16 通道音频。用于麦克风阵列和多通道系统。

阶段二：外设与通信协议

Tickless Idle

FreeRTOS 的低功耗模式。空闲时暂停 1ms 的 SysTick 中断，进入 Sleep 模式。唤醒后补偿流失的时间。在 DMA 音频流运行时仍可工作。

阶段六：系统级工程

Underrun / Overrun

欠载/过载。Underrun = DAC 没及时拿到数据（播放断音），Overrun = ADC 数据没及时读走（录音丢数据）。都会导致咔嗒声。音频系统的头号敌人。

阶段三：DMA 数据引擎

Zephyr / Device Tree / Kconfig

Zephyr 是 Linux 基金会的 RTOS。Device Tree 用声明式语法定义硬件（类似 Linux），Kconfig 用菜单选择启用哪些功能。替代 CubeMX 的手写初始化代码。
