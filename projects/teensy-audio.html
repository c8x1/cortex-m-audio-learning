Audio Graph 设计模式

Teensy Audio Library 的核心创新是 "音频图"（Audio Graph）。每个 DSP 模块是一个节点，节点之间通过连接传输音频数据块。用户在 GUI 中拖线连接模块，框架自动调度执行顺序。

源码走读：AudioStream 基类

AudioStream 是所有 DSP 模块的基类。它管理输入/输出连接、音频块缓冲区和更新调度。

AudioStream.h — 音频流基类
// Teensy Audio 的核心常量
#define AUDIO_BLOCK_SAMPLES  128   // ① 每个音频块 128 采样
#define AUDIO_SAMPLE_RATE    44100 // 默认 44.1kHz
// 块时间 = 128 / 44100 = 2.9ms

class AudioStream {
public:
    // ② 构造函数 — 声明输入端口数量
    AudioStream(uint8_t num_inputs, const uint8_t *input_list) {
        num_inputs_ = num_inputs;
        // 分配输入连接槽位
        for (int i = 0; i < num_inputs; i++) {
            inputQueue[i] = NULL;
        }
        // 注册到全局更新链表
        next_update = first_update;
        first_update = this;
    }

    // ③ 虚函数 — 子类实现具体 DSP
    virtual void update(void) = 0;

    // ④ 静态方法 — 框架调度所有模块
    static void update_all(void) {
        // 遍历链表，按连接顺序调用每个模块的 update()
        for (AudioStream *p = first_update; p; p = p->next_update) {
            p->update();
        }
    }

protected:
    // ⑤ 音频块内存池 — 静态分配，避免 malloc
    static audio_block_t *allocate(void);
    static void release(audio_block_t *block);

    // 获取输入连接的数据块
    audio_block_t *receiveReadOnly(uint8_t input_index);
    audio_block_t *receiveWritable(uint8_t input_index);

    // ⑥ 传输数据到下游连接
    void transmit(audio_block_t *block, uint8_t output_index = 0);

private:
    uint8_t num_inputs_;
    audio_block_t *inputQueue[AUDIO_MAX_INPUTS];
    AudioConnection *inputConnections[AUDIO_MAX_INPUTS];
    AudioStream *next_update;  // 链表指针
    static AudioStream *first_update;
};

// ⑦ 音频块结构 — 128 个 int16 采样
typedef struct {
    int16_t data[AUDIO_BLOCK_SAMPLES];  // 256 bytes
    uint8_t ref_count;                   // 引用计数
    uint8_t memory_pool_index;
} audio_block_t;
① 128 采样/块：在 44.1kHz 下约 2.9ms。这个大小经过优化：足够小保持低延迟，足够大让 DMA 高效传输。所有模块处理相同大小的块。
③ update() 是核心：每个模块实现自己的 update()。框架每 2.9ms 调用一次。类似 DaisySP 的 Process()，但处理整块数据而不是单采样。
⑤ 内存池：不用 malloc/free（碎片化风险），预分配固定数量的 audio_block_t。引用计数管理生命周期。这是嵌入式实时系统的标准内存管理策略。
⑦ int16 数据格式：使用 16-bit Q15 而不是 float。因为 Teensy 3.x 的 M4 没有 DSP 优化用 float 更快。不过 Teensy 4.x 的 M7 有 FPU，float 会更快。

源码走读：AudioConnection — 模块连接

AudioConnection.h — 连接两个模块
// ① AudioConnection 连接源模块的输出到目标模块的输入
class AudioConnection {
public:
    AudioConnection(AudioStream &source, AudioStream &destination) :
        src(source), dst(destination), src_index(0), dest_index(0) {
        // ② 注册连接 — 自动插入目标模块的输入队列
        destination.inputConnections[0] = this;
        // 添加到连接链表
        next = first_connection;
        first_connection = this;
    }

    // 带端口号的连接（多输入/多输出模块）
    AudioConnection(AudioStream &source, uint8_t source_output,
                    AudioStream &destination, uint8_t dest_input) :
        src(source), dst(destination),
        src_index(source_output), dest_index(dest_input) {
    }

private:
    AudioStream &src;
    AudioStream &dst;
    uint8_t src_index;
    uint8_t dest_index;
    AudioConnection *next;
    static AudioConnection *first_connection;
};

// ③ 使用方式 — 在全局作用域声明连接
AudioInputI2S          i2s_in;                    // 麦克风输入
AudioFilterBiquad      eq_filter;                 // EQ
AudioOutputI2S         i2s_out;                   // 扬声器输出
AudioConnection        patchCord1(i2s_in, 0, eq_filter, 0);  // 输入 → EQ
AudioConnection        patchCord2(eq_filter, 0, i2s_out, 0); // EQ → 输出
①② 声明式连接：连接在全局作用域创建（C++ 构造函数自动注册）。不需要手动调用 connect()。框架启动时自动建立所有连接。这种 "声明式" 设计让音频图一目了然。
③ patchCord 是连接线。每条连接线在全局创建，框架自动管理数据流。这和 Max/MSP、Pure Data 等图形化音频编程环境的设计理念一致。

学到了什么

框架设计

  • • Audio Graph 模式：模块 + 连接线
  • • 统一 update() 接口
  • • 固定块大小（128 采样）
  • • 内存池避免碎片化
  • • 引用计数管理数据块生命周期

与 DaisySP/Plaits 对比

  • • Teensy: 块处理 (128 samples) + int16
  • • DaisySP: 单采样 + float32
  • • Plaits: 小块 (12 samples) + float32
  • • Teensy 的框架级抽象最复杂
  • • Plaits 的底层优化最极致
