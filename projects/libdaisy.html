源码走读：SaiHandle — SAI/I2S 驱动

这是 libDaisy 最核心的代码。SaiHandle 封装了 STM32 SAI 外设的配置、DMA 双缓冲和中断回调，是音频数据流的起点。

src/per/sai.cpp — SAI 初始化与 DMA 配置 
// SaiHandle 类定义 (简化)
class SaiHandle {
public:
    struct Config {
        int         dev_index;       // SAI1 or SAI2
        SAI_MODE    mode;            // Master/Slave
        SAI_DIR     direction;       // TX/RX
        int         sample_rate;     // 48000, 96000, etc
        int         bit_depth;       // 16, 24, 32
        int         block_size;      // 音频块大小
    };

    // ① 初始化 — 配置 SAI 外设 + DMA + 时钟
    void Init(const Config& config);

    // ② 启动 DMA 音频流
    void StartDma(int32_t* buffer, size_t size,
                  SaiHandle::CallbackFunction cb);

private:
    SAI_HandleTypeDef hsai_;
    DMA_HandleTypeDef hdma_;
};

void SaiHandle::Init(const Config& config) {
    // ③ 使能 SAI 时钟
    if (config.dev_index == 1) {
        __HAL_RCC_SAI1_CLK_ENABLE();
    }

    // ④ 配置 SAI 为 I2S 主模式
    hsai_.Instance = SAI1_Block_A;
    hsai_.Init.AudioMode = SAI_MODEMASTER_TX;  // 主发送
    hsai_.Init.Synchro   = SAI_ASYNCHRONOUS;   // 异步模式
    hsai_.Init.OutputDrive = SAI_OUTPUTDRIVE_ENABLE;

    // ⑤ I2S 协议参数
    hsai_.Init.InitProtocol = SAI_I2S_STANDARD;
    hsai_.Init.DataSize     = SAI_DATASIZE_24BIT;
    hsai_.Init.FirstBit     = SAI_FIRSTBIT_MSB;
    hsai_.Init.FrameSize    = SAI_FRAMESIZE_32;

    // ⑥ 配置 DMA — Ping-Pong 双缓冲的关键
    hdma_.Instance                 = DMA1_Stream1;
    hdma_.Init.Channel             = DMA_CHANNEL_0;
    hdma_.Init.Direction           = DMA_MEMORY_TO_PERIPH;  // RAM → SAI
    hdma_.Init.PeriphInc           = DMA_PINC_DISABLE;
    hdma_.Init.MemInc              = DMA_MINC_ENABLE;       // 内存地址递增
    hdma_.Init.PeriphDataAlignment = DMA_PDATAALIGN_WORD;
    hdma_.Init.MemDataAlignment    = DMA_MDATAALIGN_WORD;
    hdma_.Init.Mode                = DMA_CIRCULAR;          // 循环模式！
    hdma_.Init.Priority            = DMA_PRIORITY_HIGH;

    HAL_DMA_Init(&hdma_);
    __HAL_LINKDMA(&hsai_, hdmatx, hdma_);

    HAL_SAI_Init(&hsai_);
}
① Init() 做了所有硬件配置：SAI 外设、DMA Stream、GPIO 复用、时钟分频。这是 libDaisy 封装得最好的地方 — 用户不需要碰任何寄存器。
② StartDma() 启动 DMA 循环传输并注册回调。回调会在半传输完成和全传输完成时被调用，实现 Ping-Pong。
③ 时钟使能是第一步。STM32 的每个外设独立时钟门控 — 不开时钟寄存器不工作。这是初学者最常忘记的事。
⑥ DMA_CIRCULAR 模式是音频的核心。DMA 到达缓冲区末尾后自动回到开头，不需要软件干预。配合半传输/全传输中断实现无缝循环。

源码走读：DMA 回调与音频流

src/per/sai.cpp — DMA 中断回调 
// DMA 半传输完成中断 — Ping 块就绪
void SaiHandle::DmaHalfCpltCallback(SAI_HandleTypeDef* hsai) {
    SaiHandle* self = (SaiHandle*)hsai->Parent;

    if (self->callback_) {
        // ① 告诉用户 "前半缓冲区可以处理了"
        // direction_ 标记这是输入还是输出
        // offset = 0 表示前半 (Ping)
        self->callback_(
            self->buff_,
            self->block_size_,
            0  // offset: 从缓冲区开头
        );
    }
}

// DMA 全传输完成中断 — Pong 块就绪
void SaiHandle::DmaCpltCallback(SAI_HandleTypeDef* hsai) {
    SaiHandle* self = (SaiHandle*)hsai->Parent;

    if (self->callback_) {
        // ② 告诉用户 "后半缓冲区可以处理了"
        // offset = block_size 表示后半 (Pong)
        self->callback_(
            self->buff_ + self->block_size_,
            self->block_size_,
            self->block_size_  // offset: 从缓冲区中间
        );
    }
}

// ③ 用户注册的回调 — 在这里做 DSP
void my_audio_callback(float* in, float* out, size_t size) {
    // libDaisy 自动把 int32 转为 float (-1.0 ~ 1.0)
    // 用户只需要处理 float 数据

    // 调用 DaisySP 的 DSP 模块
    for (size_t i = 0; i < size; i += 2) {
        // 立体声：每 2 个 sample 是左+右
        out[i]     = osc_.Process() * 0.5f;  // 左声道
        out[i + 1] = out[i];                  // 右声道
    }
}
① HalfCplt = Ping：DMA 正在写后半缓冲区，前半安全可读。offset=0 告诉用户从前半开始。
② Cplt = Pong：DMA 循环回前半，后半安全可读。offset=block_size 跳到后半。
③ libDaisy 的便利：它把底层的 int32 DMA 数据自动转换为 float (-1.0~1.0)，用户回调只处理 float。这比直接操作原始 DMA 缓冲区方便很多。

源码走读：AK4556 Codec 驱动

Codec 芯片（如 AK4556）负责 ADC/DAC。MCU 通过 I2C 配置它，通过 I2S 传输音频数据。

src/dev/codec_ak4556.h — Codec 初始化 
class Ak4556 {
public:
    struct Config {
        I2CHandle i2c;       // ① I2C 总线（用于配置 Codec 寄存器）
        uint8_t    address;  // Codec I2C 地址
        Pin        reset;    // ② Codec 复位引脚（GPIO 控制）
        bool       master;   // ③ MCU 是主还是从
    };

    void Init(const Config& config) {
        config_ = config;

        // ④ 硬件复位 Codec — 拉低 reset 引脚
        config_.reset.Init(Pin::Mode::OUTPUT);
        config_.reset.Write(0);   // Reset = LOW
        System::Delay(1);         // 保持 1ms
        config_.reset.Write(1);   // 释放复位
        System::Delay(10);        // 等待 Codec 就绪

        // ⑤ 通过 I2C 写 Codec 寄存器
        // AK4556 的关键寄存器：
        WriteReg(0x00, 0x07);  // Power Management: 使能 DAC + ADC
        WriteReg(0x01, 0x03);  // PLL 设置: 256FS (48kHz × 256 = 12.288MHz)
        WriteReg(0x02, 0x00);  // Format: I2S, 24-bit
        WriteReg(0x03, 0x00);  // DAC 输出: 不 mute
    }

private:
    void WriteReg(uint8_t reg, uint8_t val) {
        uint8_t data[2] = { reg, val };
        config_.i2c.Transmit(config_.address, data, 2);
    }
};
① I2C 总线只用于配置 Codec 参数（采样率、格式、增益），不传音频数据。音频数据通过 I2S（SAI 外设）传输。
② GPIO 复位：Codec 上电后需要硬件复位（reset 引脚拉低再拉高）。libDaisy 用 GPIO 控制。这是音频设备启动的标准流程。
⑤ I2C 寄存器写入对照 AK4556 数据手册理解每一位。PLL 配置决定了 MCLK 和采样率的关系。这里的 256FS 是标准配置。
逐行精读

Ping-Pong DMA 回调值得逐行理解

libDaisy 的 SAI + DMA Ping-Pong 实现是音频驱动层的黄金参考。HalfCplt / FullCplt 回调的时序、buffer 指针切换、Cache 维护——每一行都有工程含义。

架构借鉴

Codec 驱动的分层设计

libDaisy 把 Codec 配置（I2C）和音频传输（SAI）严格分离。这种控制面/数据面分离的架构模式在所有音频系统中通用。
