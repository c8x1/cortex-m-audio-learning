双核架构

nRF5340 是双核 Cortex-M33：应用核跑应用逻辑 + 网络核跑 BLE 协议栈。两核通过 IPC（核间通信）传递音频数据。

源码走读：audio_system.c — 音频管线

src/audio_system.c — 音频系统初始化与数据流
// nRF5340 Audio 的核心数据流

// ① 音频系统初始化
int audio_system_init(void) {
    // 初始化 I2S/PCM 外设 — 连接外部 Codec
    ret = i2s_slave_driver_init();
    if (ret) return ret;

    // 初始化 LC3 编解码器
    ret = lc3_codec_init();
    if (ret) return ret;

    // 初始化 BLE ISO 通道
    ret = ble_iso_init();
    if (ret) return ret;

    // ② 启动音频数据通路
    ret = audio_datapath_start();
    return ret;
}

// ③ 音频数据通路 — 从麦克风到 BLE 发送
void audio_datapath_tx_handler(void) {
    // 从 I2S DMA 获取麦克风 PCM 数据
    int16_t pcm_buf[FRAME_SIZE];  // 例：48kHz × 10ms = 480 samples
    i2s_rx_get(pcm_buf, FRAME_SIZE);

    // ④ LC3 编码 — PCM → 压缩码流
    // LC3 是 BLE LE Audio 的标准编解码器
    // 48kHz × 10ms 帧 → 约 120 bytes（码率 ~96kbps）
    uint8_t lc3_buf[LC3_FRAME_BYTES];
    lc3_encoder_process(pcm_buf, lc3_buf);

    // ⑤ 通过 IPC 发送到 Network Core
    // 使用共享内存（nRF5340 的 Net/Core 共享 RAM 区域）
    ipc_data_send(lc3_buf, LC3_FRAME_BYTES, ISO_CHANNEL_TX);

    // Network Core 的 BLE ISO 通道会自动发送
}

// ⑥ BLE 接收数据通路 — 从 BLE 到扬声器
void audio_datapath_rx_handler(uint8_t *iso_data, size_t len) {
    // LC3 解码 — 压缩码流 → PCM
    int16_t pcm_buf[FRAME_SIZE];
    lc3_decoder_process(iso_data, len, pcm_buf);

    // DSP 处理（EQ、音量等）
    audio_post_process(pcm_buf, FRAME_SIZE);

    // ⑦ 通过 I2S DMA 输出到 Codec
    i2s_tx_put(pcm_buf, FRAME_SIZE);
}
① 模块化初始化：nRF5340 Audio 按功能模块初始化 — I2S、LC3、BLE ISO。每个模块独立，便于测试和替换。
④ LC3 编码器是 BLE LE Audio 的核心。它是 Opus 的继任者，专为低延迟蓝牙音频设计。48kHz × 10ms 帧压缩到 ~96kbps，延迟仅 7.5-20ms。
⑤ IPC 核间通信：应用核和网络核通过共享内存交换数据。nRF5340 有专门的 IPC 外设和共享 RAM 区域。数据交换零拷贝 — 两核直接操作同一块内存。
⑦ I2S DMA：和 libDaisy 类似的 Ping-Pong DMA 模式，但这里用 Zephyr 的 I2S 驱动 API 而不是直接操作寄存器。Zephyr 的驱动抽象隐藏了硬件细节。

BLE LE Audio 关键概念

LE Audio 新特性

  • • ISO 通道：同步数据通道，保证时序
  • • LC3 编解码：比 SBC 质量好 50%+ 同码率
  • • BIS/CIS：广播/单播音频流
  • • 7.5ms 帧：比经典蓝牙延迟低 5-10 倍

智能眼镜应用

  • • CIS（单播）：耳机/眼镜 ↔ 手机
  • • BIS（广播）：共享音频流
  • • Auracast：公共场合音频广播
  • • 多流：同时传左耳 + 右耳
