startup.cpp — 启动代码

这是 cortexm/baremetal 项目的核心文件。用 C++ 编写启动代码是可行的——只要不依赖未初始化的全局对象（因为 C++ 运行时还没准备好）。

📁 startup.cpp C++ 裸机启动文件 
// 外部链接符号 — 由链接脚本定义
extern "C" {                             ①
    extern uint32_t _estack;             // 栈顶 (RAM 末尾)
    extern uint32_t _sidata;             // .data 在 Flash 中的起始
    extern uint32_t _sdata;              // .data 在 RAM 中的起始
    extern uint32_t _edata;              // .data 在 RAM 中的末尾
    extern uint32_t _sbss;               // .bss 起始
    extern uint32_t _ebss;               // .bss 末尾

    void Reset_Handler(void);            // 复位入口
    void Default_Handler(void);          // 默认中断处理
}

// main 函数声明
extern "C" int main(void);

// 默认处理函数 — 无限循环 (可加断点调试)
void Default_Handler(void) {             ②
    for (;;);
}

// 向量表定义
__attribute__((section(".isr_vector")))   ③
const void (*const vector_table[])(void) = {
    (void *)&_estack,                    // 初始 SP
    Reset_Handler,                       // Reset
    NMI_Handler,                         // NMI
    HardFault_Handler,                   // HardFault
    Default_Handler,                     // MemManage
    Default_Handler,                     // BusFault
    Default_Handler,                     // UsageFault
    nullptr, nullptr, nullptr, nullptr,  // 保留
    SVC_Handler,                         // SVCall
    DebugMon_Handler,                    // Debug Monitor
    nullptr,                             // 保留
    PendSV_Handler,                      ④
    SysTick_Handler,                     ⑤
    // 外部中断从此开始...
};

// 复位处理函数 — 上电后第一段执行代码
void Reset_Handler(void) {               ⑥
    uint32_t *src, *dst;

    // 1. 复制 .data 段: Flash → RAM
    src = &_sidata;                      ⑦
    dst = &_sdata;                       ⑧
    while (dst < &_edata) {
        *dst++ = *src++;                 ⑨
    }

    // 2. 清零 .bss 段
    dst = &_sbss;
    while (dst < &_ebss) {
        *dst++ = 0;                      ⑩
    }

    // 3. 调用 main()
    main();                              ⑪

    // 4. main 不应返回，但以防万一
    for (;;);
}
①extern "C" 禁止 C++ 名称修饰（name mangling）。链接脚本中的符号名是纯 C 形式的，所以必须用 C 链接约定来引用
②所有未专门处理的中断/异常都指向这个函数。实际项目中可在此加入 LED 闪烁或断点，方便调试意外中断
③__attribute__((section(".isr_vector"))) 确保向量表放在 Flash 最前面。const 表示放在只读区域
④PendSV 是可挂起的系统调用中断。FreeRTOS 将其优先级设为最低（0xFF），在所有中断处理完后用它做任务上下文切换
⑤SysTick 是 24 位递减计数器，挂在 Cortex-M 核心内部。典型配置为 1ms 中断一次，驱动 OS 时钟节拍
⑥CPU 复位后自动从向量表偏移 0x04 处读取此函数地址并跳转。此时栈已由硬件设置好
⑦_sidata 指向 Flash 中 .data 段的加载地址（LMA = Load Memory Address）。这是编译时确定的常量
⑧_sdata 指向 RAM 中 .data 段的运行地址（VMA = Virtual Memory Address）。程序运行时访问的是这个地址
⑨按 4 字节（uint32_t）逐字复制，比字节复制更高效。实际编译器可能优化为 LDM/STM 批量传送指令
⑩.bss 段只有大小信息没有初始值，所以不需要 Flash 空间。启动代码只需将对应 RAM 区域清零
⑪从上电到此处，经历了：硬件设 SP → 读 Reset_Handler → 复制 .data → 清零 .bss → 现在 main() 终于有了正确的运行环境

链接脚本 — 内存布局定义

链接脚本是启动代码的另一半。它定义了 _sdata、_edata 等符号的实际地址，让启动代码知道要复制多少数据、清零多大区域。

📁 stm32f4.ld 链接脚本 (简化) 
/* 目标芯片内存映射 */
MEMORY
{
    FLASH (rx)  : ORIGIN = 0x08000000, LENGTH = 1024K  ①
    RAM   (rwx) : ORIGIN = 0x20000000, LENGTH = 128K   ②
}

_estack = ORIGIN(RAM) + LENGTH(RAM);     ③

SECTIONS
{
    .isr_vector : {                       ④
        . = ALIGN(4);
        KEEP(*(.isr_vector))
    } > FLASH

    .text : {
        *(.text)
        *(.text.*)
        KEEP(*(.init))
        KEEP(*(.fini))
    } > FLASH

    .rodata : {
        *(.rodata)
        *(.rodata.*)
    } > FLASH

    _sidata = LOADADDR(.data);            ⑤

    .data : {
        _sdata = .;                       ⑥
        *(.data)
        *(.data.*)
        _edata = .;                       ⑦
    } > RAM AT > FLASH                    ⑧

    .bss : {
        _sbss = .;
        *(.bss)
        *(.bss.*)
        *(COMMON)
        _ebss = .;
    } > RAM

    /DISCARD/ : {
        *(.ARM.attributes)
    }
}
①STM32 的 Flash 从 0x08000000 开始（Cortex-M 的 Aliased to Code 区域）。不同芯片此地址不同，但 Cortex-M 架构规定复位时从 0x00000000 或此地址读取向量表
②SRAM 区域固定从 0x20000000 开始（Cortex-M 内存映射规范）。有些芯片有多个 RAM 块（如 SRAM1 + SRAM2），可以定义多个区域
③栈向下生长（满递减栈），所以栈顶设在 RAM 最高地址。初始化后 SP = 0x20020000（128K RAM 的末尾）
④向量表必须放在 Flash 的最前面（最低地址），因为 CPU 上电后从地址 0x08000000 开始读取向量表
⑤LOADADDR(.data) 返回 .data 段的加载地址（LMA），即 .data 的初始值在 Flash 中的存储位置。启动代码用 _sidata 作为复制源
⑥_sdata = . 记录当前位置（VMA），即 .data 段在 RAM 中的起始地址。启动代码用 _sdata 作为复制目标
⑦_edata = . 记录 .data 段的结束位置。复制字节数 = _edata - _sdata
⑧> RAM AT > FLASH 是链接脚本的核心语法：运行地址（VMA）在 RAM 中，但二进制文件中这段数据存储在 Flash 里。启动代码的复制操作就是把数据从 LMA 搬到 VMA

本节要点

  • 1. 启动代码可以用 C++ 编写，只要用 extern "C" 保护与链接脚本交互的符号
  • 2. 向量表中的 Default_Handler 是所有未处理中断的安全网，调试时可在此设断点捕获异常
  • 3. _sidata / _sdata / _edata 是链接脚本和启动代码之间的"契约接口"
  • 4. > RAM AT > FLASH 语法实现了 .data 段的"双地址"：存在 Flash 但运行在 RAM
  • 5. 栈顶设在 RAM 末尾（向下生长），堆从 .bss 之后向上生长，两者相向而行
