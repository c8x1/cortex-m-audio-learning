源码走读：引擎渲染循环

dsp/engine.cc — 核心渲染循环
// Plaits 的渲染循环 — 块处理（不是单采样）
// 每次处理 12 个采样（称为一个 "chunk"）
// 为什么是 12？为了平衡延迟和缓存效率

const int kBlockSize = 12;  // ① 块大小

void Engine::Render(const Patch& patch,
                    const Modulations& modulations,
                    float* out,
                    float* aux,
                    size_t size) {
    // ② 预处理参数平滑 — 避免参数突变导致咔嗒声
    float parameters[kNumParameters];
    for (int i = 0; i < kNumParameters; ++i) {
        float target = patch.parameters[i];
        // ③ 单极点低通滤波器做参数平滑
        // 系数 0.005f = 非常慢的平滑（~200ms 时间常数）
        smoothed_parameters_[i] += 0.005f * (target - smoothed_parameters_[i]);
        parameters[i] = smoothed_parameters_[i];
    }

    // ④ 调用当前引擎的 Render 方法
    // Plaits 支持 8 种引擎，通过 patch.engine 选择
    switch (active_engine_) {
        case ENGINE_WAVESHAPING:
            waveshaping_engine_->Render(parameters, out, aux, size);
            break;
        case ENGINE_WAVETABLE:
            wavetable_engine_->Render(parameters, out, aux, size);
            break;
        // ... 其他引擎
    }

    // ⑤ 后处理：应用 LFO 调制
    ProcessModulation(modulations, out, aux, size);

    // ⑥ 硬削波保护 — 确保输出不超过 [-1.0, 1.0]
    for (size_t i = 0; i < size; ++i) {
        out[i] = stmlib::SoftLimit(out[i]);  // 软削波
        aux[i] = stmlib::SoftLimit(aux[i]);
    }
}
① kBlockSize = 12：Plaits 使用 12 采样块处理（不是单采样也不是大块）。12 是精心选择的：足够小保持低延迟（0.25ms @ 48kHz），足够大利用缓存局部性。
③ 参数平滑是嵌入式音频的关键技术。用户旋动旋钮时参数突变，直接应用会产生咔嗒声。用单极点低通滤波器平滑参数变化是标准解决方案。
⑤⑥ 软削波：不用硬削波（out = min(1, max(-1, out))），而用 tanh 曲线。听起来更温暖，减少高频失真成分。

源码走读：CMSIS-DSP 内联优化

Plaits 大量使用 ARM CMSIS 内联函数直接操作 FPU 寄存器，这是极致优化的代表。

dsp/oscillator/sine.h — 优化的正弦波生成
// Plaits 的正弦波 — 使用多项式逼近，比 sinf() 快 10 倍
class SineOscillator {
public:
    SineOscillator() { }
    ~SineOscillator() { }

    void Init() { phase_ = 0.0f; }

    // ① 单采样处理 — 返回正弦值
    inline float Process(float frequency) {
        // ② 使用 CMSIS 内联函数 — 单周期 FPU 操作
        float phase = phase_;
        phase += frequency;
        if (phase >= 1.0f) phase -= 1.0f;
        phase_ = phase;

        // ③ 多项式逼近正弦波 (0~1 相位 → -1~+1 输出)
        // 比 sinf(2πx) 快很多，精度足够（误差 < 0.1%）
        return ThisBlepBlep(phase);  // 多项式实现
    }

    // ④ 批量处理 12 个采样 — 利用流水线
    void ProcessBlock(float frequency, float* out, size_t size) {
        // 手动展开循环，减少分支预测失败
        while (size -= 4 >= 0) {
            // 每次处理 4 个采样
            out[0] = Process(frequency);
            out[1] = Process(frequency);
            out[2] = Process(frequency);
            out[3] = Process(frequency);
            out += 4;
        }
    }

private:
    // ⑤ 多项式逼近 — 不调用任何库函数
    static inline float ThisBlepBlep(float t) {
        // 3 次多项式逼近 sin(2πt) 在 [0,1) 区间
        // 精度: 最大误差 0.0009 (-61dB)
        float s = 1.0f - t - t;  // 映射到 [-1, 1]
        s *= s;                    // s^2
        s *= s;                    // s^4
        return (s * s * (s * 6.0f - 15.0f) + 10.0f) * s;
    }

    float phase_;
};

// ⑥ Plaits 的内存对齐 — 确保 DMA 和 cache 友好
// Plaits 全局使用这个宏
#define MAKE_INTEGRAL_FRACTIONAL(x) \
    int x##_integral = static_cast(x); \
    float x##_fractional = x - static_cast(x##_integral)
①② inline + CMSIS：Plaits 几乎所有 Process() 函数都是 inline 的。编译器会内联展开，避免函数调用开销（保存/恢复寄存器 ~10 周期）。
③⑤ 多项式逼近：不使用标准库 sinf()（~30 周期），用多项式逼近（~5 周期）。牺牲微小精度换取 6 倍速度提升。这是嵌入式 DSP 的常用技巧。
④ 循环展开：每次处理 4 个采样，减少循环判断次数。M4F 的 3 级流水线在分支预测失败时要清空流水线（3 周期损失）。
⑥ 整数/小数分离宏：用于波表查找。整数部分定位波表位置，小数部分用于线性插值。这是嵌入式波表合成的标准模式。
